VideoEngager Design System

A comprehensive design system built with CSS custom properties, providing consistency across the VideoEngager platform. This token-based architecture enables theming, accessibility, and maintainability.

Overview

The VideoEngager Design System is built on a foundation of design tokens—named entities that store visual design attributes. These tokens power the entire UI, ensuring consistency and enabling theming capabilities.

Core Principles

Consistency

Uniform design language across all components and pages

Maintainability

Centralized tokens make global changes simple and reliable

Accessibility

Built with contrast, motion preferences, and focus states in mind

Flexibility

Theming system allows for different visual expressions

Token Architecture

The design system uses CSS custom properties (variables) organized into semantic categories. Tokens are defined in tokens.css and applied through tokens-bridge.css.

Token Naming Convention

Tokens follow the pattern: --category-property-modifier

/* Category */
--color-brand-500: #009EDB;

/* Property */
--font-family: Inter, sans-serif;

/* Modifier */
--shadow-md: 0 0 16px rgba(0,0,0,0.37);

File Structure

File Purpose
tokens.css Defines all design tokens (CSS custom properties)
tokens-bridge.css Applies tokens to actual UI components
theme-*.css Theme-specific token overrides

Theming System

The design system supports multiple themes through CSS custom property overrides.

How to Apply a Theme

Add a data attribute to the HTML element to switch themes:

<html data-theme="acme">
  <!-- Content -->
</html>

Available Themes

Theme Definitions

/* Default theme (in tokens.css) */
:root {
  --color-brand-500: #009EDB;
  --ring-video: #0b7cff;
  --color-text: #e6f3ff;
}

/* Acme theme (in theme-acme.css) */
[data-theme="acme"] {
  --color-brand-500: #4A90E2;
  --ring-video: #4A90E2;
  --color-text: #eaf2ff;
}

/* Zen theme (in tokens.css) */
[data-theme="zen"] {
  --color-brand-500: #00B8A9;
  --ring-video: #00B8A9;
  --color-text: #e8fffb;
}

Implementation

To use the design system in your project, include the following CSS files in your HTML:

<link rel="stylesheet" href="tokens.css">
<link rel="stylesheet" href="tokens-bridge.css">
<!-- Optional theme -->
<link rel="stylesheet" href="theme-acme.css">

Using Tokens in CSS

Reference design tokens in your custom CSS:

.custom-component {
  background-color: var(--color-surface);
  color: var(--color-text);
  padding: var(--space-4);
  border-radius: var(--radius-lg);
  box-shadow: var(--shadow-md);
  border: 1px solid var(--color-border);
}

.custom-component h2 {
  color: var(--color-text-strong);
  margin-bottom: var(--space-3);
}

Using Tokens in JavaScript

Access design tokens in JavaScript when needed:

// Get a token value
const brandColor = getComputedStyle(document.documentElement)
  .getPropertyValue('--color-brand-500');

// Set a token value (for dynamic theming)
document.documentElement.style.setProperty(
  '--color-brand-500', 
  '#newColor'
);

Best Practices

Follow these guidelines when working with the design system:

Do's and Don'ts

✅ Do

Use design tokens for all style values

Follow the established naming conventions

Test themes for accessibility and contrast

❌ Don't

Use hard-coded values instead of tokens

Create new tokens without consulting the system

Override tokens with !important

Extending the System

When you need to add new tokens:

  1. Add the token to tokens.css following naming conventions
  2. Document the token's purpose and usage
  3. Update the bridge file if the token maps to a component
  4. Test across all supported themes

Accessibility

The design system includes accessibility considerations:

  • Color contrast ratios meet WCAG AA standards
  • Respects prefers-reduced-motion settings
  • Focus states are clearly visible
  • Semantic HTML is encouraged for proper screen reader support
